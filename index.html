01

Stock Portfolio Tracker

Python Javascript
View on GitHub

Overview

A Python-based trading portfolio simulator that lets you simulate trading using fictional market data while tracking holdings, cash balance, and daily performance.

The app includes core functions such as buying and selling assets, editing holdings, viewing simulated market prices, and generating performance reports.

Key Features

  • Save & Load Portfolios — Portfolio data is persistently stored and retrieved using JSON file handling, allowing sessions to be saved and resumed at any time.
  • Simulated Market Updates — Prices update dynamically each trading day within a ±5% range, replicating real-world market volatility in a controlled environment.
  • Transaction History & Daily Summaries — Every buy and sell action is logged, with daily summaries generated to provide a clear overview of portfolio performance over time.

Challenges & Learnings

This project strengthened my Python fundamentals in data structuring and file management. It also gave me a better appreciation for how real trading systems handle dynamic data, and is something I plan to expand with a graphical interface and real market data in the future.

02

Tasklist

A console based tasklist.

Python Javascript HTML/CSS
View on GitHub

Overview

A command-line task manager built in Python that helps users stay on top of their daily workload. Tasks are stored with descriptions, deadlines, and statuses — and the program lets you add, view, complete, delete, filter, and update them all from a simple text interface.

Key Features

  • Add tasks with optional deadlines — accepts a description and an optional DD-MM-YYYY deadline, with validation to catch bad date formats
  • View all tasks at a glance — displays each task numbered with a completion checkbox, current status, and due date if set
  • Mark tasks as done — records the exact completion timestamp and updates the task's status automatically
  • Delete tasks — removes a task by its index with bounds-checking to prevent invalid deletions
  • Filter by status — view only completed, outstanding, overdue, or not-yet-overdue tasks using today's date as the reference point
  • pdate task status manually — set any custom status string on a task beyond the default in progress / completed states

Challenges & Learnings

Building this from scratch meant picking up new Python concepts along the way. One example was the enumerate() function, used to number tasks when displaying them — something unfamiliar at the start. Working through those gaps independently made completing the project a meaningful learning experience.

04

Currency Converter

A simple, console based currency converter

Python Javascript
View on GitHub

Overview

A command-line SGD currency converter built in Python. It converts Singapore Dollars into six currencies using a built-in exchange rate dataset, displays conversion tables for common SGD amounts and lets users convert any specific amount on demand all through a simple numbered menu.

Key Features

  • Individual currency table — prints a formatted SGD-to-currency table across 20 preset amounts (1, 2, 3 … up to 1,000) for a chosen currency
  • All currency tables at once — loops through all six supported currencies (USD, THB, CNY, JPY, GBP, MYR) and prints a table for each
  • Convert a specific amount — prompts the user for any SGD value and target currency, then outputs the exact converted result with input validation
  • Timestamped session header — displays the current date and time on launch so the user knows when the exchange rates were last set

Challenges & Learnings

Building this from scratch meant figuring out how to structure a menu-driven program, handle bad user input , and format output which made completing this project extremely satisfying as it taught me many new key concepts in programming and taught me how to work with numbers on code.

Watch Website

A terminal based watch buying process coded using python

Python Javascript
View on GitHub

Overview

A command-line luxury watch store built in Python. Users are able to pick from three brands and learn a bit of their history, browse available models with prices, and add multiple watches to a cart that tallies a running total before checkout.

Challenges & Learnings

This project was built without much prior knowledge on programming and had thought me many key concepts. One of which is nested dictionaries which I have had to use many times over school projects.

06

Shopping Cart

A terminal based shopping coded using python

Python Javascript
View on GitHub

Overview

A command-line shopping cart built in Python, and the direct foundation for the watch website project. It introduced the core e-commerce logic product listings, a persistent cart, and a full checkout flow.

Key Features

  • Product catalogue & cart — browse four luxury items, add multiple products by ID with custom quantities, and view a live cart with a running subtotal
  • Flexible cart removal — remove a specific quantity of an item or wipe it entirely with all, with checks to prevent removing more than what's in the cart
  • Multi-method checkout — supports Credit/Debit Card, PayNow, and Cash on Delivery, each with its own input flow and validation
  • Card masking & input validation — card numbers are masked to show only the last four digits; phone and card lengths are validated before the order is confirmed

Challenges & Learnings

The biggest challenge was building the checkout flow from scratch. Handling three different payment paths, validating card and phone number lengths, and writing the card masking logic using .join() and string slicing. Structuring the program around functions and a shared cart dictionary also taught me how data persists across different parts of the program.
